RALPH - Autonomous Development Methodology

What it is: A structured framework for autonomous AI-driven development using Claude Code CLI. RALPH stands for a repeatable loop that transforms requirements into production code through atomic task execution with zero context loss.

The Loop: PRD (requirements) ← progress.txt (execution state) ← DONE.txt (completed tasks) ← Archive (version history). Each cycle: read PRD, check progress, execute one atomic task, update progress, verify, repeat.

Key Principles: Every task is atomic and independently verifiable. Progress is persisted to file after every step - no reliance on session memory. The agent reads its own previous output to maintain continuity. PRD is the single source of truth. Archive folder preserves full version history.

File Structure: ralph.bat (launcher), ralph-prompt.txt (system prompt injected into Claude Code), SKILL.md (project-specific technical context), PRD.md (requirements), progress.txt (execution journal), DONE.txt (completion log), archive/ (version snapshots).

Why it works: Claude Code has no persistent memory between sessions. RALPH solves this by externalizing all state to the filesystem. The result is a coding agent that can be interrupted, resumed, or handed off without losing a single line of context.

Claude Code - Development Environment & Configuration

Role definition: Claude Code is configured as a DevOps & Automation Engineer (not "AI Engineer"). The environment enforces production-first, infrastructure-builder mindset with Hebrew-first communication.

Global CLAUDE.md: A master configuration file loaded automatically into every Claude Code session. Contains identity definition, tech stack preferences, project conventions, work guidelines, and known issues. This ensures consistent behavior across all projects without re-explaining context.

Per-project CLAUDE.md: Each project directory can have its own CLAUDE.md that extends the global config with project-specific instructions, architecture decisions, and active TODOs.

SKILL.md pattern: For complex projects, a SKILL.md file provides deep technical context - architecture diagrams (text-based), API contracts, database schemas, integration points, and deployment configs. Claude Code reads this before executing any task.

Work guidelines enforced: Test before committing. Use conventional commits. Docker Compose for all services. Never hardcode secrets. Check existing code before creating new files. Verify with user before destructive operations.

Key conventions: Python 3.11+ with type hints. FastAPI for APIs. Celery + Redis for async. PostgreSQL for persistence. All configs via environment variables. Makefile for common commands.

MCP Servers - Connected Intelligence Layer

What is MCP: Model Context Protocol - a standard for connecting AI models to external tools and data sources. David's Claude Code environment has multiple MCP servers configured, giving the AI agent direct access to real tools and data.

Active MCP Servers:

Obsidian REST API Read/write access to personal knowledge vault (1,400+ notes). Enables the AI to search, retrieve, and update documentation in real-time. Used for memory continuity across sessions.

Filesystem MCP Direct read/write access to the development workspace. The AI can create files, read configs, modify code, and navigate project structures without manual copy-paste.

GitHub MCP Repository management, issue creation, PR operations, and code search across all repos. Enables autonomous Git workflows.

Sequential Thinking A meta-cognitive MCP that forces structured reasoning: break problem into steps, evaluate each step, revise if needed. Prevents the AI from rushing to solutions without analysis.

Context7 Library documentation lookup. Fetches up-to-date docs for any framework or library, reducing hallucination risk when generating code.

Architecture principle: MCP servers turn Claude Code from a text generator into an autonomous agent with real-world tool access. Combined with RALPH methodology, this creates a development loop where the AI can read requirements, write code, test it, commit to Git, and update documentation - all without human intervention for routine tasks.
